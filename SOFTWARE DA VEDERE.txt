---------------- INFO SUI TIPI DI SERVER ---------------------------------


- Rivedi i termini:
	- web server ed esempi (esempio: apache server, che restituisce pagine web (es:. statiche in HTML), 
	  o pagine web dinamiche che prevedono l'esecuzione di servizi mediante supporti runtime locali (es:. php) 
          oppure integrare funzionalità di Application Server). Non può 
	- application server ( servlet container (tomcat) esegue applicativi java enterprise che NON comportano esecuzione di EJB; + funzionalità EJB container
	  quindi esegue applicazioni in java o altro linguaggio, e permette di gestire pagine dinamiche

- Tomcat fornisce anche funzionalità di web server, quindi può restituire pagine web statiche, così da non dover integrare un web server apache apposito.
